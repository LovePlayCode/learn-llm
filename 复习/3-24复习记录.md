# 3-24 复习记录

## 复习内容一：垃圾回收（第7天间隔检索）

### 掌握的内容

- **标记-清除两阶段**：标记阶段遍历对象引用图标记可达对象，清除阶段释放未标记对象。
- **GC 触发时机 bug**：正在使用但尚未挂靠到对象图上的临时对象会被误回收。原因是还没来得及建立引用关系。解决方案：分配前 push 到 VM 值栈，操作完成后 pop。
- **自适应阈值**：新阈值 = 当前存活内存 × 增长因子。基本掌握。
- **Open Upvalue 与 Closed 状态**：知道 closed 不是独立类型，而是 ObjUpvalue 的一种状态。

### 需要加强

- **三色标记完整流程**：灰色的定义不够精确。正确理解：白色（未发现）→ 灰色（已发现为可达，但子对象待遍历，入栈）→ 弹出遍历子对象 → 黑色（自身和子对象都处理完毕）。
- **GC 根集合6个来源**：值栈和 CallFrame 栈需要分开记。完整6个：① VM 值栈 ② CallFrame 栈 ③ 全局变量表 ④ 字符串驻留表 ⑤ Open Upvalue 链表 ⑥ 编译器对象链。
- **Closed Upvalue 的 location 指向**：不是泛泛的"堆"，而是指向 ObjUpvalue **自身结构体内部的 closed 字段**。关闭操作 = 把栈上的值复制到 closed 字段 + location 指针重定向到自身 closed 字段。
- **递归导致栈溢出的原因**：不是互相引用，而是对象引用链太深时递归深度过大。灰色栈将调用栈的压力转移到堆上的数组。

## 复习内容二：AI工程第一章（第2天间隔检索）

### 掌握的内容

- **预训练→微调→RLHF 三步链路核心作用**：预训练建立知识底座（补全机器），微调实现格式转换（支持问答），RLHF让回答更好更安全。
- **三种适配技术优先顺序**：提示工程 → RAG → 微调，从简单便宜到复杂昂贵。
- **RAG 的必要性**：公司内部知识不在模型预训练数据中，只用提示工程会产生幻觉，需要 RAG 引入外部知识库。
- **AI工程 vs ML工程工作流**：AI工程是产品→数据→模型，ML工程是数据→模型→产品。

### 需要加强

- **三阶段数据特点（量 vs 质）**：预训练靠量（~10TB，质量一般），微调靠质（~10万条，人工精心标注，质量极高）。
- **RAG 精确描述**：检索到的相关文档作为**上下文（context）**喂给模型，模型基于上下文生成回答——不是"继续优化"。
- **AI工程不需要ML知识的原因**：AI工程师主要工作在应用开发层（提示工程、上下文构建、评估），用现成模型 API，只有应用层搞不定才往模型开发层深入。

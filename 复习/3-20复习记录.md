## 零样本思维链（Zero-shot CoT）

### 掌握的内容

- **定义和触发语**：在 prompt 末尾添加 "Let's think step by step" 触发模型逐步推理。
- **工作原理**：触发语 → 模型输出中间推理步骤 → 每步输出成为下一步的上下文输入 → 复杂任务被拆解成多个小步骤 → 每步难度降低 → 准确率提升。
- **与少样本CoT的区别**：零样本靠一句触发语，少样本靠提供包含推理过程的示例。
- **局限性**：简单任务不需要思维链，强制使用会画蛇添足——不必要的中间步骤可能引入错误，错误会沿推理链传递放大。

### 需要加强

- 工作原理的完整因果链条需要多复述，第一次回答时只说了一半（"每步输出成为下一步输入"），缺少"拆解任务、降低单步难度"这个关键点。

## 少样本学习（Few-shot Learning）

### 掌握的内容

- **定义**：在 prompt 中提供少量输入→输出示例，让模型识别任务模式并模仿。模型不更新参数，不是真正"学习"，而是模式匹配和模仿。
- **完整提示结构**：任务指令 → 示例对（输入→输出） → 新的输入。
- **与少样本CoT的关系**：少样本思维链是少样本学习的特殊形式，示例从"输入→输出"变成"输入→推理过程→输出"，多加了推理过程。

### 需要加强

- **上下文内学习（In-context Learning）**：连续两次忘记这个术语。核心绑定记忆：**少样本学习的核心机制 = 上下文内学习 = 不更新参数，只做模式匹配**。

## Transformer

### 掌握的内容

- **核心机制**：自注意力机制（Self-Attention）。
- **解决了 RNN 的两个关键问题**：
  1. **长距离依赖问题**：RNN 逐token串行处理，第1个词的信息到第1000个词需经过999步传递，信息逐步衰减。自注意力机制中每个词同时和所有其他词直接计算关联度，任意两个位置一步直达，无衰减。
  2. **无法并行计算**：RNN 每步依赖上一步输出，必须串行。自注意力中所有词可同时处理，支持并行计算。
- **一步直达的原理**（本次复习中自己想通的）：自注意力并行计算时，每个词同时和序列中所有其他词直接计算关系，不需要中间词传递，所以不管距离多远都是一次计算。

### 需要加强

- **与提示词工程的关系比喻**没记牢，需要强化：Transformer（架构）= 车的引擎和构造，决定**能力上限**；提示词工程 = 驾驶技巧，决定**能力发挥**。**架构决定能力上限，提示词工程决定能力发挥。**

# 3-19 复习记录

## 垃圾回收（Crafting Interpreters 第26章）

### 掌握的内容

- **标记-清除算法**：分为标记阶段和清除阶段。标记阶段从根集合出发遍历对象图，找到可达对象；清除阶段释放不可达对象的内存。
- **灰色栈机制**：标记阶段使用一个灰色对象数组（worklist）管理待处理对象，避免递归导致栈溢出。根对象染灰入栈 → 弹出灰色对象遍历其引用 → 引用的白色对象染灰入栈，自身变黑 → 重复直到栈为空。
- **GC 触发时机的 bug**：在分配操作中触发 GC 时，正在使用但尚未挂靠到对象图上的临时对象可能被误回收。解决方法：在分配前先 push 到 VM 值栈使其可达，操作完成后 pop。
- **GC 触发条件**：已分配内存超过阈值时触发，GC 后新阈值 = 当前已用内存 x 增长因子（自适应策略）。
- **Open Upvalue**：闭包捕获的仍在栈上的变量，挂在 VM 上统一管理（因为多个闭包可能捕获同一个变量）。关闭时 location 指针从栈上重定向到自身 closed 字段，是同一个 ObjUpvalue 对象的两种状态。

## Transformer 注意力机制

### 第一次复习几乎全忘，重新输入后掌握

### 掌握的内容

- **交叉注意力**：关注输入序列和输出序列之间的关系，在输入序列中找到与输出序列更契合的 token。
- **自注意力**：在同一个序列内部，找到每个 token 之间的关系。
- **解决的核心问题**：RNN 处理长序列时记忆丢失（长距离依赖问题）。RNN 是逐词串行处理，距离越远信息衰减越严重；自注意力机制中任意两个 token 之间直接连接，不管距离多远都是"一步到位"。

### 经验教训

- 第一次学完隔夜几乎全忘，说明输入时缺少主动加工。**下次学新内容，看完立刻复述，不要隔夜。**

## 大语言模型文本补全

### 掌握的内容

- **核心流程**：用户输入提示词 → 大语言模型分析上下文序列 → 在后续可能的词中选择概率最高的词 → 逐 token 生成，每次生成一个 token 拼回输入序列再预测下一个（自回归生成）。

### 补充知识（教材未覆盖）

- **停止机制**：模型生成时遇到特殊的结束标记（EOS token）时停止生成。
- **temperature 参数**：可以控制选词的随机性，并不总是选概率最高的词。以后深入学习时会遇到。

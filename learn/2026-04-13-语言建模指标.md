# 语言建模指标（3.2节）

**学习日期：** 2026-04-13

## 核心概念

### A层（必须掌握）

1. **困惑度（Perplexity/PPL）**：模型预测下一个token的不确定程度。困惑度为N，意味着模型相当于在N个等概率选项中选择。
2. **困惑度越低 = 模型预测越准确**
3. **困惑度的影响因素：**
   - 数据结构化程度越高 → 困惑度越低
   - 词表越大 → 困惑度越高
   - 上下文长度越长 → 困惑度越低
4. **困惑度的局限：** 困惑度只衡量"预测下一个token"的能力，不能完全代表模型在实际任务上的表现。后训练（RLHF等）可能导致困惑度升高，但任务表现更好。

### B层（理解思路）

5. **熵（Entropy）：** 一个token平均携带的信息量
6. **交叉熵（Cross-Entropy）：** 模型预测下一个token的难度
7. **困惑度 = 交叉熵的指数形式**（PPL = 2^H）
8. 交叉熵、BPC、BPB、困惑度四者密切相关，可互相转换

### C层（用时查阅）

9. 熵的数学定义、KL散度公式
10. BPC与BPB的具体换算
11. 困惑度的精确计算公式

## 知识地图

```
语言建模指标
├── 熵 (Entropy) ——每个token平均携带多少信息
│   └── 交叉熵 (Cross-Entropy) ——模型预测的难度
│       ├── BPC / BPB ——交叉熵的不同单位换算
│       └── 困惑度 (PPL) ——交叉熵的指数形式 ⭐最常用
│           ├── 影响因素：数据结构化程度、词表大小、上下文长度
│           ├── 应用：模型比较、数据污染检测、异常文本检测
│           └── 局限：后训练后PPL可能变高，不能完全代表任务表现
```

## 应用场景

- **模型选型：** 比较不同模型的PPL
- **数据污染检测：** 如果模型对某段文本PPL异常低，可能该文本在训练数据中
- **异常文本检测：** 无意义或不寻常的文本PPL会很高

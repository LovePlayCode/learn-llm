## 核心知识点

### 1. AI Agent 的定义
AI Agent 是让大型语言模型（LLM）真正**做事**的系统——通过赋予它工具和知识，让它操控世界，而不仅仅是响应提示。

关键理解：LLM 本身只能生成文本，Agent 系统让 LLM 拥有了"手和脚"，能够执行实际操作（搜索数据库、调用 API、发送消息等）。

### 2. Agent 的三要素
- **环境（Environment）** — Agent 所工作的空间（如旅游预订平台）
- **传感器（Sensor）** — Agent 感知环境的方式，即"怎么看"（如查酒店空房、查机票价格）
- **执行器（Actuator）** — Agent 执行操作的方式，即"怎么做"（如预订房间、发确认邮件、取消订单）

这三者解耦设计，类似前端组件化思想，好处是可独立替换、可扩展、易调试。

### 3. Agent vs 普通 LLM 对话
| 维度 | 普通 LLM（如 ChatGPT 对话） | AI Agent |
|------|---------------------------|----------|
| 核心能力 | 生成文本 | 感知 → 决策 → 行动（循环） |
| 工具使用 | 无 | 可调用外部工具和 API |
| 记忆 | 仅当前对话窗口 | 短期记忆（当前对话上下文）+ 长期记忆（数据库、用户偏好） |
| 执行操作 | 不能 | 能（通过执行器） |

### 4. 工具访问
- Agent 能用什么工具取决于：(1) 运行环境 (2) 开发者赋予的能力
- 工具是 LLM 的"手和脚"，让它从"只会说话"变成"能做事"

### 5. 两种记忆
- **短期记忆** — 当前对话上下文（类比前端组件的 state）
- **长期记忆** — 客户数据库、历史交互、用户偏好（类比数据库持久化数据）

### 6. 前端类比
Agent 的运行机制类似浏览器**事件循环**：
- 传感器感知环境变化 ≈ 产生事件
- LLM 推理决策 ≈ 处理逻辑
- 执行器操作环境 ≈ 执行事件

关键差异：事件循环是确定性的，Agent 中的 LLM 推理是**不确定的**，会根据上下文自主决策。

# 2026-03-26 学习记录：Transformer 模型架构（2.2节）

## 学习目的

作为前端工程师，理解 LLM 底层原理，达到"知其所以然"的程度。

## 知识分层

### A层（核心，需深入理解）

1. **seq2seq 的两个问题**
  - 信息瓶颈：解码器只能拿到编码器的最终隐藏状态（像只根据书的摘要回答问题）
  - 顺序处理：RNN 必须按顺序处理 token，无法并行，速度慢
2. **注意力机制 QKV**
  - Q（查询）：当前在找什么信息
  - K（键）：每个 token 的索引/标签
  - V（值）：每个 token 的实际内容
  - Q 和 K 点积 → 注意力分数（相关性）→ softmax → 加权求和 V
3. **Transformer 并行能力**
  - 输入端：可以并行处理所有输入 token（抛弃了 RNN）
  - 输出端：仍然是自回归，逐个生成，无法并行

### B层（理解思路）

1. **多头注意力**：将 QKV 拆分成多个子向量，允许模型同时关注不同部分
2. **Transformer 模块组成**：注意力模块 + MLP模块（前馈层），堆叠 N 层
3. **推理两阶段**：预填充（并行处理输入）→ 解码（逐个生成输出）

### C层（查阅即可）

1. 注意力公式：Attention(Q,K,V) = softmax(QK^T/√d)V
2. 权重矩阵维度：d_model 相关
3. 嵌入层、位置嵌入、解嵌入层
4. 激活函数：ReLU、GELU

## 关键理解

- 三个不同的权重矩阵让模型学到不同的表示空间
- 缩放因子防止点积过大导致 softmax 饱和
- Transformer 抛弃 RNN 后失去位置信息，用位置编码弥补

## 薄弱点

- 初次复述时遗漏了"信息瓶颈"问题，需加强对 seq2seq 两个问题的区分

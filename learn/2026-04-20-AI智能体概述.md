**学习日期**：2026-04-20
**来源章节**：6.2.1 智能体概述
**学习目的**：准备自己动手开发一个智能体

---

## 一、核心定义（A 层 · 必背）

> **智能体（agent）= 任何能够【感知】其环境并【作用于】环境的事物。**

这个定义里只有**一个核心要素：环境**。

### ⚠️ 最反直觉的一点
- 定义里 **没有"智能"**
- 定义里 **没有"AI"、没有"LLM"、没有"大脑"**
- 一个只会"温度 > 26°C 就开空调"的恒温器，按这个定义也是智能体

**这个定义把"智能体"和"AI"解耦了**——智能体是一种**架构**，AI 只是让这个架构变聪明的**可选升级**。

### 工程意义
1. 可以**先搭骨架**（环境接口、动作集合、循环机制），**再决定**大脑用什么
2. 可以**替换大脑**而不重写系统（今天 GPT-4，明天 Claude，后天本地小模型）
3. 可以**分层开发**——先用 if-else 跑通，再接 AI

---

## 二、关键概念

### 2.1 环境（environment）
智能体所处的场景，由应用决定：
- 游戏 agent → 游戏
- 爬虫 agent → 互联网
- 烹饪机器人 → 厨房
- SWE-agent → 带终端和文件系统的计算机
- RAG 系统 → 知识库 / 数据库

### 2.2 动作集合（set of actions） vs 工具（tools）

| 动作（actions） | 工具（tools） |
|---|---|
| 智能体**本质上能做的事**（动词清单） | 动作的**实现手段 / 扩展** |
| 像"一个人的能力"（能走、能抓、能说） | 像"他手里拿的东西"（锤子、笔、手机） |

**关键原文**：
> "AI 智能体能够执行的动作集合，通过其可使用的工具得到扩展。"

👉 **工程启示**：代码里真正要定义的是**动作集合**，工具只是实现动作的方式。

### 2.3 环境和工具的"强依赖"（双向）
- **环境 → 决定工具**：象棋环境里智能体只能走合法棋步
- **工具 → 限制环境**：只会游泳的机器人只能在水环境活动

---

## 三、智能体的工作模式

**核心认知：智能体不是一个函数，是一个循环。**

```
感知环境 → AI 决策 → 执行动作 → 环境反馈 → 再感知 → 再决策 ……
```

和编程中的**事件循环 / 游戏主循环 / REPL** 本质相似：
- 都是"监听→反应→再监听"的**无限循环**
- 不是一锤子买卖

---

## 四、智能体 vs 普通 LLM 调用

| 维度 | 单次 LLM 问答 | 智能体 |
|---|---|---|
| 环境 | 用户 prompt（一段文字） | 具体场景（数据库、系统、互联网等） |
| 动作 | 只能回答 | 感知 / 决策 / 工具调用 / 反思 / 回答 |
| 步骤 | 一步 | 多步循环 |
| 信息获取 | 被动接收 prompt | **主动探索**（查数据库、搜网页、读文件） |

**本质差异**：智能体把 LLM 从"被动回答者"变成了"主动探索者"。

## 五、典型案例

### 5.1 SWE-agent（编程智能体）
- 环境：带终端和文件系统的计算机
- 动作集合：浏览代码仓库 / 搜索文件 / 查看文件 / 编辑代码行

### 5.2 Kitty Vogue RAG 智能体（8 步示例）
查询："预测未来三个月 Fruity Fedora 的销售收入"

1. 推理如何完成任务
2. 调用 SQL 查询生成工具（获取过去 5 年销售数据）
3. 调用 SQL 执行工具
4. 判断数据不足（缺失值），需要过去营销活动信息
5. 调用 SQL 查询生成工具（查营销活动）
6. 调用 SQL 查询执行工具
7. 判断信息已足够，生成预测
8. 任务完成

👉 **关键观察**：智能体的推理过程以 AI 回复消息的内容呈现。

### 5.3 ChatGPT（带联网搜索 + 代码执行）也是智能体
- 感知环境 ✅（通过搜索结果 / 代码输出感知）
- 作用于环境 ✅（生成代码执行、发起搜索请求）
- 动作集合：搜索网页、执行 Python、生成图像

## 六、为什么智能体需要更强的模型？

### 6.1 错误累积效应
- 每步 95% 准确率：
  - 10 步后 ≈ **60%**
  - 100 步后 ≈ **0.6%**
- 随步骤数增加，整体准确率指数级下降

### 6.2 更高的风险
能够使用工具 → 能执行影响更大的任务 → 错误的后果更严重

## 七、动手开发智能体前必问的两个问题

1. **我要开发的这个智能体，它将在什么样的【环境】里运行？**
2. **我要给这个智能体定义哪些【动作】？（以及用什么【工具】实现这些动作？）**

这两个问题没想清楚，代码一行都写不了。

## 八、学习过程中的认知纠偏

### 误区 1（学前）："智能体 = 能完成一系列任务的助手"
→ 太模糊，任何程序都能套。**真正定义**要求"感知+作用于环境"这个循环架构。

### 误区 2："智能体 = 带 AI 的程序"
→ 错。恒温器不带 AI 也是智能体。**智能体是架构，AI 是可选升级**。

### 误区 3："环境 + 工具 + AI = 智能体"
→ 错。定义里只有**环境**一个核心要素，其他都是展开。

### 误区 4："动作 = 工具"
→ 错。动作是**本质能力**，工具是**实现手段**，工具扩展动作。

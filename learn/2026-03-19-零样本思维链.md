# 2026-03-19 零样本思维链（Zero-shot Chain of Thought）

## 核心概念

- **定义：** 零样本思维链是提示词工程的一种策略，通过在 prompt 末尾添加一句简短的触发语，引导大语言模型逐步输出推理过程，从而提升复杂任务的回答准确率。
- **经典触发语：** `Let's think step by step.`
- **来源：** Kojima 等人 2022 年论文

## 工作原理

- 大语言模型是逐 token 生成的
- 触发语引导模型输出**逐步展开的推理步骤**（第一步、第二步、第三步……）
- 每一步的输出成为下一步生成的**上下文输入**
- 模型用自己生成的中间步骤文本引导自己走向正确答案

## 与少样本思维链（Few-shot CoT）的区别

| | 零样本思维链 | 少样本思维链 |
|---|---|---|
| **是否提供示例** | 不提供任何推理示例 | 在 prompt 中提供具体的推理示例 |
| **触发方式** | 仅靠一句触发语 | 靠示例引导模型模仿推理过程 |

## 适用场景与局限性

- **适用：** 多步推理任务（数学题、逻辑推理等）
- **不适用：** 常识类简单问题（模型已在训练数据中学过，不需要分步推理）
- **注意：** 简单任务强制使用思维链可能画蛇添足，不必要的中间推理步骤反而可能引入错误

## 知识连接

- 与递归的对比：
  - **相似点：** 上一步输出作为下一步输入
  - **不同点：** 递归每一步调用自身做同一件事且有终止条件；思维链每一步做不同的推理，无显式终止条件
  - 更准确地说，零样本思维链是一个**顺序的推理链条（chain）**，而非递归

# 2026-03-19 少样本学习（Few-shot Learning）

## 核心概念

- **定义：** 少样本学习是提示词工程的一种策略，通过在 prompt 中提供少量的输入→输出示例对，让模型识别任务模式，并按照相同模式处理新的输入。
- **核心机制：** 上下文内学习（In-context Learning）——模型不更新参数，而是从示例中识别和模仿模式。
- **注意：** 模型并非真正"学习"（没有参数更新），而是在做**模式匹配和模仿**。

## 完整的少样本提示结构

1. **任务指令**（告诉模型要做什么）
2. **示例对**（输入 → 输出，2个以上）
3. **新的输入**（需要模型处理的内容）

## 提示词策略光谱

| 策略 | 示例数量 | 说明 |
|---|---|---|
| **零样本提示（Zero-shot）** | 0 个 | 只给任务指令，不给示例 |
| **单样本提示（One-shot）** | 1 个 | 给一个示例 |
| **少样本提示（Few-shot）** | 2+ 个 | 给多个示例 |

## 与少样本思维链（Few-shot CoT）的关系

- **少样本学习**的示例：输入 → 输出
- **少样本思维链**的示例：输入 → **推理过程** → 输出
- 少样本思维链是少样本学习的一种**特殊形式**，示例中额外包含了推理步骤

# 2026-03-19 Transformer

## 核心概念

- **定义：** Transformer 是一种模型架构（architecture），核心机制是自注意力机制（Self-Attention），解决了 RNN 的两个关键问题：长距离依赖问题和无法并行计算的问题。
- **来源：** Vaswani 等人 2017 年论文 "Attention Is All You Need"
- **适用范围：** 不仅用于大语言模型，也广泛用于图像（ViT）、语音等领域

## RNN 的两个关键问题

1. **长距离依赖问题：** RNN 逐步处理序列，信息层层传递，第 1000 个词要拿到第 1 个词的信息需要经过 999 步，信息逐步衰减丢失。
2. **无法并行计算：** RNN 每一步的输入依赖上一步的隐藏状态（hidden state）输出，形成严格的顺序依赖，必须一个 token 一个 token 按序处理，无法并行。

## 自注意力机制（Self-Attention）的优势

1. **一步直达：** 序列中任意两个位置的信息可以一步直接交互，不需要中间传递，从根本上解决长距离依赖问题。
2. **可并行计算：** 每个词的计算不依赖其他词的计算结果，所有词可以同时处理，大幅提升训练速度。

## 知识连接

- **架构 vs 提示词工程的关系：**
  - Transformer（架构）= 车的引擎和构造，决定**能力上限**
  - 提示词工程（零样本思维链、少样本学习等）= 驾驶技巧，决定**能力发挥**
  - **架构决定能力上限，提示词工程决定能力发挥**
